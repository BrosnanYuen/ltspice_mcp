# note1.md

## 1) Web research summary for active op-amp band-pass (40 Hz to 20 kHz)

Research basis:
- TI, *Op Amps for Everyone*, Chapter 16 (SLOA088 / SLOD006):
  - first-order RC cutoff relationship: `f_c = 1/(2*pi*R*C)` for RC sections (shown in low-pass and high-pass derivations)
  - for wide-band band-pass filters, the simplest implementation is a high-pass and a low-pass in series
  - active filters use op-amps to isolate/cascade stages and avoid loading effects

Design approach chosen (wide-band requirement 40 Hz to 20 kHz):
- First-order HP stage at 40 Hz
- First-order LP stage at 20 kHz
- Non-inverting Universal op-amp stage (gain = 2 V/V)
- Supplies: +12 V and -12 V rails

Why this topology:
- The TI text explicitly calls HP+LP-in-series the simplest wide-band band-pass approach.
- It maps directly to the required edge frequencies and runs robustly in LTspice with UniversalOpAmp1.

## 2) Calculated values

Used formula:
- `R = 1/(2*pi*f_c*C)`

Chosen capacitors and computed resistors:
- HP target `f_L = 40 Hz`, choose `C_HP = 100 nF`
  - `R_HP = 1/(2*pi*40*100e-9) = 39,788.74 ohm` -> use `39.8k`
- LP target `f_H = 20 kHz`, choose `C_LP = 1 nF`
  - `R_LP = 1/(2*pi*20000*1e-9) = 7,957.75 ohm` -> use `7.96k`
- Op-amp gain set with non-inverting feedback:
  - `A_v = 1 + RFB/RG`
  - `RFB = 10k`, `RG = 10k` -> `A_v = 2`

## 4) What makes `.net` valid vs invalid (observed in this run)

### Valid patterns that worked

1. Proper first-line title/comment and proper `.end`
- Works with:
  - line 1 title/comment
  - simulation directives before `.end`

2. Correct element keywords and node/value fields
- Working line starters in this design:
  - `V`, `R`, `C`, `X`, `.ac`, `.meas`, `.lib`, `.options`, `.end`

3. Correct Universal op-amp usage
- Working op-amp instance:
  - `XU1 NBP NNEG VCC VEE OUT level1 Avol=1Meg GBW=10Meg ...`
- Required library include for this model name:
  - `.lib UniversalOpAmp1.lib`

4. Correct dual rails
- `VCC VCC 0 12` creates +12 V rail
- `VEE 0 VEE 12` creates -12 V rail at node `VEE`

5. Correct AC source and AC analysis
- Input source includes `AC 1`
- `.ac dec 400 1 1Meg` runs successful sweep

6. Correct AC measurement form
- Working:
  - `.meas AC FL WHEN mag(V(OUT))=GMAX/sqrt(2) RISE=1`
  - `.meas AC FH WHEN mag(V(OUT))=GMAX/sqrt(2) FALL=last`

### Invalid patterns that failed

I created and ran three invalid files:

1. Invalid device keyword prefix (`Y`)
- File: `invalid_keyword.net`
- Failing line: `Y1 IN OUT 1k`
- Failure output (`invalid_keyword_2.fail`):
  - `Expected nodes here.`

2. Missing library for op-amp subcircuit name
- File: `invalid_missing_lib.net`
- Failing line:
  - `XU1 NBP NNEG VCC VEE OUT level1 ...`
  - without `.lib UniversalOpAmp1.lib`
- Failure output (`invalid_missing_lib_3.fail`):
  - `This sub-circuit name is not defined.` (points to `level1`)

3. Invalid numeric text tail with strict number parsing
- File: `invalid_number_tail.net`
- Failing line: `R1 IN OUT 4Farads`
- With `.options reject_number_tails`
- Failure output (`invalid_number_tail_4.fail`):
  - `Syntax error. (Note: option "reject_number_tails" is set.)`

## 5) MCP server API/tool calls that worked vs failed

Server used: `ltspice_mcp`

### Worked in this run
- `runtime_info` -> confirmed LTspice and Wine configured
- `execute(api_name="SimRunner", ...)` -> created runner object
- `execute(api_name="SimRunner.run", ...)` -> started netlist runs
- `execute(api_name="wait_results", ...)` -> returned output paths (`.raw/.log`) or `.fail`
- `execute(api_name="RawRead", ...)` -> loaded `.raw`
- `execute(api_name="traces_to_csv", ...)` -> exported CSV trace file
- `execute_status` -> completed async operations

### Failed in this run
- `execute(api_name="run", inputs={"object_name":"runner_bp"})`
  - Result from `execute_status`: `{"status":"invalid input!","operation":"execute"}`

## 6) Keywords that worked vs keywords that failed

### Worked keywords (from successful netlist)
- Device/instance prefixes: `V`, `R`, `C`, `X`
- Dot directives: `.ac`, `.meas`, `.lib`, `.options`, `.end`
